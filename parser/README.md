# ⚙️ SSSK Parser (ядро системи)

Цей модуль відповідає за автоматичний збір, розпізнавання та структурування даних про відключення для Старокостянтинівської громади.

## 🛠️ Функціонал та логіка
Робот працює автономно через **GitHub Actions** і виконує наступні кроки:
1. **Збір (Monitoring):** Одночасна перевірка сайту `hoe.com.ua` та офіційного Telegram-каналу.
2. **Аналіз змін:** Порівняння хеш-сум зображень. Якщо картинка не змінила свій хеш — обробка припиняється для економії ресурсів та уникнення дублікатів.
3. **Розпізнавання (OCR):** Використання Tesseract для вилучення тексту з графіків та пошуку ключових слів ("ОНОВЛЕНО", "ТЕРМІНОВО").
4. **Структурування:** Збереження даних у `unified_schedules.json` для подальшого відображення в PWA.

## 🕒 Режими роботи (Scheduling)
Парсер має інтелектуальну систему запусків для балансу між актуальністю та лімітами GitHub:

- **🚀 Режим Пік (19:00 - 23:00):** Перевірка кожні **15 хвилин**. Це "гарячий час", коли обленерго публікує графіки на наступний день.
- **😴 Стандартний режим:** Перевірка кожні **5 годин** у спокійний час (ніч або день без змін).
- **⚡ Режим Override (Ручний):** Можливість форсувати перевірку через змінні оточення:
  - `OVERRIDE_SOURCE`: вибір джерела (`site`, `telegram` або `both`).
  - `OVERRIDE_INTERVAL`: встановлення кастомного інтервалу в хвилинах.
  - `OVERRIDE_DURATION`: тривалість дії ручного режиму (в годинах).

## 📂 Структура даних
- `src/` — вихідний код (Python).
- `data/state.json` — поточний стан (останні хеші, час запуску, метадані).
- `data/unified_schedules.json` — база даних розпізнаних графіків.
- `data/raw_site/` та `data/raw_telegram/` — "Data Lake" з оригінальними картинками для історії.
- `data/logs/` — технічні логи роботи (логування рівня INFO/ERROR).

## 🛠️ Технології
- **Python 3.10**
- **Tesseract OCR** (розпізнавання тексту з навчанням під українську мову)
- **Pillow** (обробка зображень перед OCR)
- **Requests & BeautifulSoup4** (парсер сайту та взаємодія з Telegram API)
- **Logging** (професійне відстеження роботи)
