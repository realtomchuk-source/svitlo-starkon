# Svitlo Starkon (Світло-Старкон) 💡

Система моніторингу графіків відключень електроенергії для мешканців Старокостянтинівської громади.

## 🚀 Про проєкт
Повністю **безсерверна (Serverless)** система на базі GitHub Actions + GitHub Pages.

## ✅ Phase 1 — Автоматичний Моніторинг (поточна)

### Функціонал:
- **Dual-Source парсинг:** Два незалежних джерела — сайт [hoe.com.ua](https://hoe.com.ua/page/pogodinni-vidkljuchennja) та Telegram-канал обленерго
- **Автоматичне виявлення змін:** Хеш-порівняння зображень графіків для миттєвого виявлення оновлень
- **OCR-розпізнавання:** Tesseract OCR з українською мовною моделлю для витягування тексту з графіків
- **Адаптивний розклад:** Два режими моніторингу — стандартний (активні відключення) та тимчасовий (відсутність відключень)
- **Telegram-алерти:** Сповіщення при виявленні нового або оновленого графіка
- **Data Lake:** Збереження сирих даних (зображення) для подальшого аналізу

### Розклад моніторингу:

**Стандартний режим** (при активних відключеннях):
| Час (EEST) | Частота |
|---|---|
| 19:00 – 00:00 (пік) | кожні 5 хв |
| 00:00 – 09:00 (ніч) | кожні 3 год |
| 09:00 – 19:00 (день) | кожні 3 год |

**Тимчасовий режим** (відсутність відключень):
| Час (EEST) | Частота |
|---|---|
| 19:00 – 23:00 (пік) | кожні 15 хв |
| Решта доби | кожні 5 год |

## 🔜 Phase 2 — Аналітика (заплановано)
- Піксельний парсинг графіків по групах 1.1–6.2
- Реконструкція розкладу в структурований JSON (`current.json`, `tomorrow.json`)
- Історія та версії графіків (`history/`, `versions/`)
- Індекс нестабільності (аналіз зміни планів протягом дня)
- Метрики: % часу відключень, найбільш нестабільна група

## 📋 Phase 3 — PWA (заплановано)
- Таймлайн по групах з візуалізацією
- Історія всіх графіків
- Статистика та порівняння
- Офлайн-режим, встановлення на телефон

## 🏗 Архітектура даних
Дані зберігаються у форматі JSON у папці `/data`:
- `state.json` — Стан парсера (хеші, мітки часу, поточне джерело)
- `unified_schedules.json` — Журнал виявлених графіків (timestamp, джерело, OCR-текст)
- `raw_site/` — Сирі зображення з сайту
- `raw_telegram/` — Сирі зображення з Telegram
- `current.json` — *(Phase 2)* Актуальний розклад по групах

## 🛠 Технологічний стек
- **Parser:** Python 3.10 (Requests, BeautifulSoup4, PIL/Pillow)
- **OCR:** Tesseract OCR + ukr.traineddata
- **CI/CD:** GitHub Actions (адаптивний cron-розклад)
- **Database:** Flat JSON files
- **Frontend:** *(Phase 3)* PWA (HTML5, Vanilla JS, CSS3)
- **Deployment:** GitHub Pages
- **Alerts:** Telegram Bot API

## ⚙️ Налаштування

### GitHub Secrets (обов'язково):
| Secret | Опис |
|---|---|
| `TELEGRAM_BOT_TOKEN` | Токен бота для Telegram-алертів |
| `TELEGRAM_CHAT_ID` | ID чату для надсилання сповіщень |

### Ручний запуск:
1. **Actions** → **SSSK Monitor** → **Run workflow**
2. Обрати джерело: `site`, `telegram`, або `both`
3. Опційно: задати форсований інтервал та тривалість
